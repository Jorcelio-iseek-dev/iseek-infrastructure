────────────────────────────────────────
1) PROPÓSITO
────────────────────────────────────────

Este documento orienta como navegar
pela documentação de infraestrutura
do projeto iSeek Automations.

Ele define:

• ordem recomendada de leitura
• finalidade de cada documento
• relação entre os documentos


────────────────────────────────────────
2) LEITURA RECOMENDADA
────────────────────────────────────────

Para entender rapidamente a infraestrutura
do projeto, recomenda-se a seguinte ordem:


PASSO 1 — Visão Geral

docs/architecture/ISEEK-INFRA-ARCHITECTURE-v1.0.txt

Descrição geral da infraestrutura,
componentes principais e arquitetura.


PASSO 2 — Estado Atual

docs/registry/entries/
ISEEK-INFRA-2026-001-infra-core-saas-baseline-v1.0.txt

Define o baseline congelado da infraestrutura.


PASSO 3 — Modelo de Risco

docs/architecture/ISEEK-INFRA-RISK-MODEL-v1.0.txt

Identifica riscos técnicos e estratégias
de mitigação.


PASSO 4 — Governança

docs/governance/ISEEK-INFRA-GOVERNANCE-CHARTER-v1.0.txt

Define princípios de governança
da infraestrutura.


PASSO 5 — Políticas Operacionais

docs/governance/ISEEK-INFRA-CHANGE-POLICY-v1.0.txt

Define como mudanças devem ser realizadas.


PASSO 6 — Registro de Decisões

docs/governance/ISEEK-INFRA-DECISION-LOG-v1.0.txt

Registro histórico de decisões técnicas.


PASSO 7 — Runbooks Operacionais

docs/runbooks/

Contém guias operacionais para execução
de tarefas específicas.
